## Highest-impact

### 1. Subagent/background-daemon reliability fixes — directly hit our observed failure modes
- **Source**: https://code.claude.com/docs/en/changelog — 2.1.200, July 3, 2026 · seen 2026-07-06
- **What it is**: Fixes for background sessions silently stopping after sleep/wake, stale `daemon.lock`
  preventing agent restarts, and — most relevant — **"subagents cut off by a rate limit before producing
  any text output returning an empty result instead of failing cleanly"** (now fails cleanly). Also:
  `SendMessage` no longer silently misroutes when a re-spawned agent reuses a previous agent's name.
- **Why it matters here**: The empty-result-on-rate-limit bug is EXACTLY the "dead/empty specialist" case
  our constitution §14a handles (retry once with a clarified work order, then escalate) — the synaipse run
  hit session-limit-killed parallel fix agents. With 2.1.200 the failure signal is clean, so §14a triggers
  reliably instead of the PM occasionally seeing an ambiguous empty YAML. No harness change needed; the
  platform now matches our assumption.
- **Recommendation**: **ignore (no change) / note** — confirms §14a; ensure Claude Code ≥ 2.1.200.
- **Status**: NEW

### 2. `AskUserQuestion` no longer auto-continues by default
- **Source**: https://code.claude.com/docs/en/changelog — 2.1.200, July 3, 2026 · seen 2026-07-06
- **What it is**: Question dialogs no longer auto-continue on idle by default (idle timeout is now an
  opt-in via `/config`).
- **Why it matters here**: The harness leans on `AskUserQuestion` for every user-gated decision (design
  ambition/direction, preset+model confirm, "what next?"). Auto-continue could previously time out past a
  taste decision; now those gates wait indefinitely for the user — strictly better for the two-level
  acceptance model. Users who WANT idle timeouts must now opt in per `/config`.
- **Recommendation**: **ignore (benign; behaviour now matches the harness's intent)**.
- **Status**: NEW

### 3. Notification hook fires for background agents (`agent_needs_input` / `agent_completed`)
- **Source**: https://code.claude.com/docs/en/changelog — 2.1.198, July 1, 2026 (in the prior window but
  not previously surfaced) · seen 2026-07-06
- **What it is**: Background agent sessions that need input or finish now fire the `Notification` hook
  with `agent_needs_input` / `agent_completed`.
- **Why it matters here**: V14a's rule ("after parallel spawns, never advance the phase before ALL
  notifications") is currently prompt-level. A small `Notification`-hook could **deterministically log**
  background-agent completions/input-requests into `project_memory/.audit/hook_events.jsonl` — making
  parallel-spawn accounting visible to `retro.py` and auditable (who finished, who stalled), at near-zero
  cost. Complements, not replaces, the prompt rule.
- **Recommendation**: **adopt (small)** · low effort — a ~30-line `notify_agent_events.py` hook +
  settings registration in both kits, next hardening round.
- **Status**: NEW

## Worth watching

### 4. Anthropic "Managed Agents": multi-agent orchestration now public beta (platform tier)
- **Source**: https://claude.com/blog/new-in-claude-managed-agents +
  https://www.mindstudio.ai/blog/code-with-claude-2026-new-agent-features (Code with Claude event, early
  July 2026) · seen 2026-07-06
- **What it is**: Platform-level productization of the lead-agent→specialist pattern: a managed lead agent
  decomposes work, delegates to specialist sub-agents (own model/prompts/tools), parallel on a shared file
  system, fully auditable in Claude Console. Public beta (with "outcomes" + memory); "dreaming" in
  research preview.
- **Why it matters here**: This is the SAME architecture the harness implements locally (PM → specialists,
  single-writer artifacts, audit trail) — validation of the design, and long-term a possible managed
  runtime for the team kits (cloud-hosted PM instead of a local session). Today it is a separate product
  tier (Console/Managed Agents, not Claude Code CLI) with no kit-level integration path.
- **Recommendation**: **watch** — revisit when Managed Agents expose custom constitutions/hooks or a
  migration path from Claude Code agents.
- **Status**: NEW
