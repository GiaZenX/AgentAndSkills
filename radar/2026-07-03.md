## Highest-impact

### 1. Subagents now run in the background by default
- **Source**: https://code.claude.com/docs/en/changelog ‚Äî 2.1.198, July 1, 2026 ¬∑ seen 2026-07-03
- **What it is**: A spawned subagent no longer blocks the parent ‚Äî the lead keeps working while the
  subagent runs and is notified on completion (previously a gradual rollout, now the default). Same
  release: subagents inherit the session's extended-thinking config; the launching agent's messages count
  as task direction but never as user approval.
- **Why it matters here**: This touches the core PM‚Üíspecialist flow. The constitution assumes the PM
  delegates one implementation task and receives the specialist's YAML back synchronously before the
  phase advances (REVIEW‚ÜíTEST‚ÜíQA‚ÜíMERGE). Background-by-default breaks that "await the YAML" assumption
  and interacts with `guard_agent_spawn.py` and the `Task` merge gate. The "agent message ‚âÝ user
  approval" rule is a helpful confirmation that the two-level acceptance model is safe.
- **Recommendation**: **watch ‚Üí verify** ¬∑ medium effort. Test the dev-team flow under 2.1.198 to
  confirm the PM still blocks on specialist YAML (or pin behaviour explicitly). If it does not, decide
  whether to embrace parallel specialists or force foreground delegation.
- **Status**: NEW

### 2. Permission rules can now gate subagent spawns declaratively (`Agent(...)`)
- **Source**: https://code.claude.com/docs/en/changelog ‚Äî `Tool(param:value)` syntax added 2.1.178
  (June 15, 2026); `Agent(type)` deny / `Agent(x,y)` allowlist enforcement fixed 2.1.186 (June 22, 2026);
  spawn evaluated by the auto-mode classifier before launch 2.1.178 ¬∑ seen 2026-07-03
- **What it is**: `settings.json` permission rules now match tool input params, e.g. `Agent(model:opus)`
  to block Opus subagents, and `Agent(type)` / `Agent(x,y)` type allow/deny rules are now actually
  enforced for named spawns.
- **Why it matters here**: `guard_agent_spawn.py` today enforces which specialists the PM may spawn (and
  the model_map) in custom Python. Part of that could move to declarative `permissions` in the kit's
  `settings.json` ‚Äî less code to maintain, enforced by the runtime rather than a hook. Good candidate to
  complement (not replace) the guard as defense-in-depth.
- **Recommendation**: **adopt (partial) / watch** ¬∑ low effort. Add `Agent(...)` allow/deny rules to the
  kit settings as a belt-and-suspenders layer; keep the hook for logic permissions can't express.
- **Status**: NEW

### 3. Hook matcher correctness fixes ‚Äî audit our matchers
- **Source**: https://code.claude.com/docs/en/changelog ‚Äî comma-separated matchers silently never firing,
  fixed 2.1.191 (June 24, 2026); hyphenated identifiers now exact-match, 2.1.195 (June 26, 2026) ¬∑
  seen 2026-07-03
- **What it is**: `"Bash,PowerShell"`-style comma matchers were silently dead; hyphenated names
  (`code-reviewer`, `mcp__brave-search`) were substring-matching. Both fixed.
- **Why it matters here**: Our kit `settings.json` matchers use pipe alternation (`Edit|Write`,
  `Agent|Task`, `Bash`, `Write`) ‚Äî none comma-separated or hyphenated, so we are **not** affected. Worth
  recording as a checked invariant: never write comma matchers; if a future MCP-scoped matcher is added,
  use `mcp__server__.*` form.
- **Recommendation**: **ignore (no change) / note** ¬∑ trivial. Audit-only; matchers are already correct.
- **Status**: NEW

### 5. Stop / SubagentStop hooks can return `additionalContext`
- **Source**: https://code.claude.com/docs/en/changelog ‚Äî 2.1.163 (June 4, 2026) ¬∑ seen 2026-07-03
- **What it is**: `Stop` and `SubagentStop` hooks may return `hookSpecificOutput.additionalContext` to
  feed the model context and keep the turn going, without being treated as a hook error.
- **Why it matters here**: Our `Stop` hook `auto_dashboard.py` currently just regenerates the dashboard.
  It could additionally hand the PM a short "dashboard changed: X" note via `additionalContext` so the PM
  reflects the update in-turn instead of silently. Same mechanism could let a future `SubagentStop` gate
  return a structured "specialist YAML incomplete" nudge.
- **Recommendation**: **watch** ¬∑ low effort. Nice-to-have polish on the dashboard/PM loop.
- **Status**: NEW

- **Why it matters here**: If we move toward parallel/background specialists (see item 1), these events

### 7. Community: the Workflow tool for deterministic many-agent orchestration
- **Source**: https://platform.claude.com/docs/en/agent-sdk/subagents and
  https://platform.claude.com/docs/en/managed-agents/multi-agent ¬∑ seen 2026-07-03
- **What it is**: Anthropic's guidance splits delegation into two tiers ‚Äî conversational **subagents**
  for a few tasks per turn, and the **Workflow tool**, which lifts orchestration into a runtime-executed
  script (deterministic, outside conversation context) for coordinating many agents. Community write-ups
  echo the "deterministic control plane + LLM reasoning nodes" pattern.
- **Why it matters here**: This harness deliberately keeps orchestration *in prose* (constitution) with
  the PM as a conversational lead. The Workflow-tool pattern is the sanctioned path if we ever want the
  phase model (0‚Üí10) executed deterministically rather than PM-narrated. Not a today change ‚Äî but it's
  the direction of travel and validates our "deterministic gates around LLM reasoning" thesis.
- **Recommendation**: **watch / ignore for now** ¬∑ high effort. Note as a possible future rearchitecture
  of the phase engine; no action this cycle.
- **Status**: NEW

### A1. All five Bash-gates are bypassable via the Windows `PowerShell` tool ‚Äî **CORRECTS item 3** ¬∑ `adopt (verify-first)` ¬∑ ~1‚Äì2 h
- **Source**: hook matcher docs (`Edit|Write` / `Edit, Write`; comma tolerance needs v2.1.191+) ‚Äî
  https://code.claude.com/docs/en/hooks ¬∑ changelog 2.1.195 `autoMode.classifyAllShell` = "route all
  **Bash/PowerShell** commands" and 2.1.191 comma-matcher example `"Bash,PowerShell"` ‚Äî
  https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md ¬∑ **direct evidence**: this Claude Code
  session exposes a distinct `PowerShell` tool alongside `Bash` ¬∑ seen 2026-07-03
- **What it is**: On Windows, Claude Code surfaces PowerShell as its own matchable shell tool (name
  `PowerShell`), separate from `Bash`. The changelog treats "Bash/PowerShell" as two first-class shells.
- **Why item 3's "we are not affected" is wrong**: item 3 only checked for *comma/hyphen* matcher bugs
  and cleared us. But the enforcement spine binds every shell gate to `"matcher": "Bash"`
  (`gate_git`, `gate_test_coverage`, `gate_memory_complete`, `gate_packaging_decision`, `gate_pipeline`),
  and each hook body **hard-exits unless the tool is Bash** ‚Äî e.g. `gate_git.py:44`
  `if data.get("tool_name") != "Bash": sys.exit(0)`. On this Windows-first harness (`install.ps1`,
  PowerShell scaffold scripts) a PM/specialist that runs `git push --force`, `git merge`, or a
  coverage-less `pytest` **through the PowerShell tool** sails past all five ‚Äî the force-push block and the
  QA-report merge gate included. The gates only *look* enforced.
- **Recommendation**: confirm the target Windows setup can issue commands via a `PowerShell` tool (it can
  here), then (a) widen every shell matcher to `"Bash|PowerShell"` ‚Äî keep the `|` form, not `"Bash,PowerShell"`
  which no-ops on CC < 2.1.191 ‚Äî and (b) relax the `tool_name != "Bash"` guard in each gate to accept
  `PowerShell`, reading `tool_input.command` identically. Add a `tools/test_hooks.py` case feeding a
  `PowerShell`-tool `git push --force` payload and asserting exit 2.
- **Status**: NEW

### A2. Sharper framing of item 5 ‚Äî `SubagentStop` as a specialist-deliverable *return-gate* ¬∑ `adopt` ¬∑ ~2‚Äì4 h
- **Source**: hooks docs ‚Äî `SubagentStop` fires "when a subagent finishes"; Stop/SubagentStop accept
  `hookSpecificOutput.additionalContext` for "non-error feedback that continues the conversation" ‚Äî
  https://code.claude.com/docs/en/hooks ¬∑ seen 2026-07-03
- **Beyond item 5 (which framed this as dashboard polish)**: neither kit has **any** `SubagentStop` hook,
  so the delegation *return path* is unenforced ‚Äî `guard_agent_spawn.py` gates the spawn but nothing gates
  the result. A `SubagentStop` gate could verify the just-finished specialist actually wrote its required
  `project_memory/` artifact (e.g. `quality-engineer` ‚Üí a `*report*.yaml`) and, if missing, hand the PM an
  `additionalContext` nudge to re-dispatch ‚Äî *without* aborting the turn. This is exactly the community
  "verification layer" (Anthropic multi-agent-coordination-patterns; Shipyard) and it's the natural safety
  net for background-by-default subagents (item 1).
- **Recommendation**: prototype `hooks/gate_subagent_deliverable.py` on `SubagentStop`, nudging via
  `additionalContext` rather than exit 2, keyed off the returning role.
- **Status**: NEW
